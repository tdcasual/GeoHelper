# Ultrawide Settings Drawer Design

**Goal:** 修复超宽屏桌面下设置抽屉过窄、表单列空间不足的问题。

## Context

线上 `2560×1440` 视口下，设置抽屉总宽固定为 `520px`，左侧导航占 `200px` 后，正文内容区只剩约 `275px`。这会让多个表单控件显得拥挤，尤其是长 URL、模型设置与多列设置块。移动端与中小屏设置抽屉当前表现正常，因此应只针对大桌面做放宽。

## Options Considered

1. 全局把设置抽屉从 `520px` 提高到 `680px`
- 优点：实现简单。
- 缺点：会影响普通桌面和窄桌面，不够精准。

2. 仅在超宽屏媒体查询下放宽抽屉，并略增导航列宽与 gap
- 优点：精准命中问题断面，风险最小。
- 缺点：需要补一小段桌面专用 CSS。

3. 改成中央 modal 而不是右侧 drawer
- 优点：理论上可利用更多空间。
- 缺点：交互变化过大，不符合当前最小修复目标。

## Recommendation

采用方案 2：
- 保持默认桌面与移动端设置抽屉不变。
- 在超宽屏桌面下，抽屉放宽到 `720px` 左右，并把导航列提升到 `220px`、列间距适度加宽。
- 只修复空间分配，不调整设置内容结构。

## Testing

- 新增 E2E：`2560×1440` 下打开设置抽屉，drawer 宽度与内容区宽度必须达到最低可读阈值。
- 继续运行既有 `fullscreen-toggle` / `geogebra-mount` 套件，确保移动端与窄屏不回归。
