# Desktop Empty State Design

**Goal:** 修复超宽屏桌面空会话时右侧聊天区过于空旷的问题，让空态更聚焦，并提供快捷模板入口。

## Context

线上 `2560×1440` 视口下，聊天消息区高度超过 `1100px`，但当前空态只有一条靠顶部的虚线提示，导致视觉重心过高、引导很弱。现有模板系统已经有默认模板与填充 draft 的交互，因此可以直接复用，不需要新建数据结构。

## Options Considered

1. 只把现有空态垂直居中
- 优点：实现最简单。
- 缺点：虽然减少空白，但仍然缺少可点击的开始路径。

2. 做桌面专用空态卡片，并复用现有模板作为快捷入口
- 优点：既修复空白感，也降低首次使用成本。
- 缺点：需要补少量结构和样式。

3. 直接展开完整模板面板
- 优点：选择更多。
- 缺点：信息量过大，和当前问题不成比例。

## Recommendation

采用方案 2：
- 仅在非 compact 桌面下，用居中的空态卡片替换顶部单行提示。
- 卡片内展示标题、简短说明和前 3 个模板快捷按钮。
- 按钮点击后直接填充 composer 并聚焦输入框，复用已有模板填充行为。
- compact/mobile 继续保留现有简洁空态，避免手机界面过度复杂。

## Testing

- 新增 E2E：`2560×1440` 下空态卡片可见、相对消息区近似垂直居中。
- 新增 E2E：点击快捷模板按钮后，composer 中出现对应 prompt。
- 继续运行现有 `fullscreen-toggle` 与 `geogebra-mount` 回归，确保不影响移动端和窄屏修复。
