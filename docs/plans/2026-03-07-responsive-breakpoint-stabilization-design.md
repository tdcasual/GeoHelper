# Responsive Breakpoint Stabilization Design

**Goal:** 修复两个已在线上复现的响应式问题：窄桌面打开历史后聊天区被压扁，以及低高度横屏设备顶部栏过高导致 GeoGebra 可视区不足。

## Context

线上证据显示，`901×600` 视口下桌面布局仍使用固定右侧聊天面板，打开历史后聊天正文仅剩约 `186px` 宽，输入区与标题区出现明显拥挤。`844×390` 低高度横屏下应用进入 compact 单面板模式，但顶部栏仍沿用较宽屏样式，高度约 `99px`，直接侵占 GeoGebra 垂直空间。

## Options Considered

1. **整体提高 compact 断点**
   - 优点：实现简单。
   - 缺点：会把大量小桌面/平板宽屏直接切到单面板模式，行为变化过大。

2. **窄桌面保留双栏，但把历史切到覆盖层**
   - 优点：只修正真正失效的子区域，不改变主布局。
   - 缺点：需要在桌面聊天壳内增加一套覆盖式历史样式。

3. **仅调小历史最小宽度和聊天宽度比例**
   - 优点：改动最小。
   - 缺点：只能缓解，不能保证可用宽度。

## Recommendation

采用 **方案 2 + 低高度横屏专用压缩样式**。

- 当桌面聊天壳宽度不足时，历史不再挤占聊天正文，而是以覆盖层形式出现在聊天面板内部。
- 引入 `short-viewport` 语义类，仅在 `innerHeight <= 500` 时压缩顶部栏、按钮和画布/对话切换器高度。
- 保持当前 GeoGebra profile 选择逻辑不变，避免额外引入工具栏/视图模式回归。

## Testing

- 新增 E2E 回归：`901×600` 打开历史后聊天正文宽度不得明显塌陷。
- 新增 E2E 回归：`844×390` 紧凑横屏顶部栏高度必须受控。
- 继续运行已有 GeoGebra mount / fullscreen / mobile overflow 回归，确保无副作用。
